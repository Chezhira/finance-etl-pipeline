# 

# \# Security Policy

# 

# \## Supported Versions

# We maintain the `main` branch and the latest tagged release (starting from v0.1.0).

# 

# \## Reporting a Vulnerability

# If you discover a security vulnerability, please report it responsibly:

# \- Open a private advisory via \*\*GitHub Security Advisories\*\*.

# \- Or email: \zahidm@chezsolutions.co.uk

# Include:

# \- Steps to reproduce the issue.

# \- Affected files or components.

# We aim to respond within \*\*5 business days\*\*.

# 

# \## Data Safety

# This project uses \*\*synthetic finance-like data only\*\*.  

# Do \*\*NOT\*\* commit:

# \- Real PII (Personally Identifiable Information).

# \- Credentials, API keys, or secrets.

# 

# \## Dependencies

# We recommend running `pip install --upgrade` regularly and checking `pip-audit` for dependency vulnerabilities.

#

# 

# Thank you for helping keep this project secure!
